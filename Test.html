數據縮放與線性降維

本應用程式將帶您深入了解單細胞轉錄組測序（scRNA-seq）分析流程中的兩大核心橋樑步驟。

分析上下文位置

在完成單細胞數據的品質控管（QC）與標準化（Normalization）後，接下來必須進行數據縮放（Scaling）與線性降維（PCA）。這兩個步驟直接決定了後續聚類（Clustering）與視覺化（UMAP/t-SNE）的準確性與有效性。

⚖️

步驟一：Scaling

消除基因間表達量的量級差異，並標準化方差，確保特徵權重平等。

📉

步驟二：PCA

通過主成分分析進行降維與降噪，提取關鍵生物學信號以供下游使用。

1. 數據縮放 (Scaling the Data)

在進行降維之前，必須對數據進行線性變換。這是為了確保所有基因在後續分析中具有平等的權重，避免高表達基因主導分析結果。

🎯 為什麼需要 Scaling？

  • 消除量級差異：某些高度表達的基因（如管家基因）數值波動極大。若不縮放，會掩蓋微小但重要的轉錄因子變化。
  • 標準化方差：使每個基因的平均值為 0，標準差為 1（Z-score 轉換）。

⚙️ 實作細節

  • 平移 (Shifting)：平均表達值歸零。
  • 縮放 (Scaling)：標準差歸一。
  • 回歸排除：可移除細胞週期、線粒體基因佔比、批次效應等干擾因素。

互動展示：Scaling 變換效果

圖示：未縮放前，高表達基因 (X軸) 的變異幅度完全掩蓋了低表達基因 (Y軸)。

2. 線性降維 (Linear Dimensional Reduction)

單細胞數據包含上萬個基因。直接在高維空間聚類會面臨「維度災難」。主成分分析 (PCA) 是最常用的線性降維方法，通過旋轉坐標軸找到能解釋最大變異的方向。

🌟 PCA 核心目的

  • 降噪：捕捉主要的生物學信號，過濾掉隨機產生的背景噪聲。
  • 特徵壓縮：將上千個變異基因壓縮成數十個相互正交（無相關性）的主成分 (PCs)。
  • 預備後續：作為 UMAP 視覺化和圖論聚類的基準。

📏 決定維度

保留多少 PCs 是分析的關鍵：

  • Elbow Plot (碎石圖)：尋找標準差下降曲線趨於平緩的「拐點」。
  • JackStraw：統計學顯著性檢定。
  • 經驗法：通常保留前 20 到 50 個 PCs。

分析流程總結與程式碼

標準工作流

1
NormalizeData
消除測序深度影響
2
FindVariableFeatures
挑選具生物學意義基因
3
ScaleData
Z-score 轉換，消除偏差
4
RunPCA
計算主成分投影
5
Determine Dimensionality
決定使用多少 PCs

常用程式碼範例 (R/Seurat)

# 1. 縮放數據 (預設僅針對高變異基因)
                                                                                                                                                                                                                                      all.genes <- rownames(pbmc)
                                                                                                                                                                                                                                      pbmc <- ScaleData(pbmc, features = all.genes)
                                                                                                                                                                                                                                      
                                                                                                                                                                                                                                      # 2. 執行 PCA
                                                                                                                                                                                                                                      pbmc <- RunPCA(pbmc, features = VariableFeatures(object = pbmc))
                                                                                                                                                                                                                                      
                                                                                                                                                                                                                                      # 3. 檢查 PCA 結果
                                                                                                                                                                                                                                      print(pbmc[["pca"]], dims = 1:5, nfeatures = 5)
                                                                                                                                                                                                                                      
                                                                                                                                                                                                                                      # 4. 視覺化 PC 分布
                                                                                                                                                                                                                                      VizDimLoadings(pbmc, dims = 1:2, reduction = "pca")
                                                                                                                                                                                                                                      DimPlot(pbmc, reduction = "pca")
                                                                                                                                                                                                                                      
                                                                                                                                                                                                                                      # 5. 決定維度 (Elbow Plot)
                                                                                                                                                                                                                                      ElbowPlot(pbmc)

💡 實務建議

  • 若樣本存在明顯細胞週期干擾，建議在 ScaleData 中使用 vars.to.regress 參數。
  • 務必檢查每個 PC 貢獻最大的基因。若前幾個 PC 全是線粒體/核糖體基因，代表 QC 階段需重新調整。
